常见Git方法

工作区、暂存区、版本区

					//工作区提交到暂存区：
					//git add -A  提交所有变化
					//git add -u  提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
					//git add .  提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件
					
					//暂存区提交到版本区：
					//git commit -m '注释'
						
					//工作区提交到版本区：-a是add的缩写
					//git commit －a －m '注释'

分支

					//git branch <本地分支名> 新建本地分支
					//git checkout <本地分支名> 切换本地分支
					//git checkout -b <本地分支名> 创建并切换本地分支
						
					//合拼分支（在master分支下）
					//git merge <本地分支名> 本地分支名合并到master分支下
						
					//删除本地分支
					//git branch -d <本地分支名>
						
					//删除远程分支
					//git push origin -d <远程分支名>
						
					//查看分支
					//git branch 查看本地分支
					//git branch -r 查看远程分支
					//git branch -a 查看所有分支
					
					//把远程分支的代码pull到本地分支
					//git pull <远程主机名> <远程分支名>:<本地分支名>
					//PS：push代码前最好先用pull更新本地代码
					
					//把本地分支代码push到远程分支
					//git push <远程主机名> <本地分支名>:<远程分支名>
					//PS：如果省略远程分支名，则表示将本地分支推送与之存在"追踪关系"的远程分支（通常两者同名），如果该远程分支不存在，则会被新建。
					//PS：远程主机名一般为origin
